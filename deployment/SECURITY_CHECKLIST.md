# Security Checklist

## أسرار وبيئة

- [ ] لا ترفع `.env.production` إلى Git.
- [ ] استخدم `SECRET_KEY` عشوائي قوي.
- [ ] لا تستخدم كلمات مرور افتراضية.
- [ ] لا تفعل `DEBUG=true` في الإنتاج.
- [ ] لا تفتح PostgreSQL للعالم الخارجي إلا عند الحاجة.
- [ ] استخدم Firewall.

## التطبيق

- [ ] CORS محدد للدومينات الفعلية فقط.
- [ ] Docs/OpenAPI معطلة في الإنتاج بسبب `DEBUG=false`.
- [ ] تحقق من صلاحيات المستخدمين.
- [ ] راقب محاولات الدخول الفاشلة.
- [ ] فعّل HTTPS عند تركيب الشهادة.

## الملفات

- [ ] اضبط حجم رفع الملفات.
- [ ] لا تسمح برفع أنواع خطرة.
- [ ] استخدم CDN أو Object Storage لاحقًا للملفات الكبيرة.

## النسخ الاحتياطي

- [ ] جدولة backup يومي.
- [ ] اختبار restore دوريًا.
- [ ] حفظ نسخ خارج السيرفر.

## Hardening المضاف

- [ ] اضبط `ALLOWED_HOSTS` على الدومينات الفعلية + localhost للـ healthcheck الداخلي.
- [ ] أبقِ `RATE_LIMIT_ENABLED=true` في الإنتاج.
- [ ] اضبط `RATE_LIMIT_LOGIN` و `RATE_LIMIT_PUBLIC_FORM` حسب حجم الزيارات.
- [ ] راقب `X-Request-ID` في logs عند أي خطأ.
- [ ] تأكد أن `/api/v1/health/ready` يرجع ready بعد تشغيل الخدمات.
