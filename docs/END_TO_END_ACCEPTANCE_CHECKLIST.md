# End-to-End Acceptance Checklist

> هذه القائمة تستخدم لاعتماد المشروع محليًا قبل اعتباره جاهزًا للانتقال لاحقًا إلى مرحلة النشر.

## 1. Backend

- [ ] `python -m compileall config platform_api scripts tests` ينجح.
- [ ] `python scripts\quality_check.py` ينجح.
- [ ] `pytest` ينجح.
- [ ] تسجيل الدخول يعمل.
- [ ] إنشاء مستخدم بكلمة مرور يعمل، وكلمة المرور لا تظهر في API.
- [ ] الصلاحيات تمنع الوصول غير المصرح به.
- [ ] كل أخطاء 400/401/403/404 ترجع `success=false`.
- [ ] استجابات النجاح ترجع `success=true`.

## 2. Admin Panel

- [ ] `pnpm run release-check` ينجح.
- [ ] `pnpm run quality` ينجح.
- [ ] `pnpm run build` ينجح.
- [ ] تسجيل الدخول يعمل.
- [ ] Dashboard يظهر الإحصائيات والتنبيهات.
- [ ] الإعدادات العامة تحفظ وتظهر في public settings.
- [ ] رفع الوسائط يعمل.
- [ ] اختيار الوسائط داخل الفورم يعمل.
- [ ] إنشاء/تعديل/حذف خدمة يعمل.
- [ ] إنشاء/تعديل/حذف نظام يعمل مع features/images/faqs.
- [ ] إنشاء/تعديل/حذف تطبيق يعمل مع download_url/download_files/screenshots.
- [ ] إنشاء/تعديل/حذف عمل يعمل مع gallery_images/technologies.
- [ ] إنشاء/تعديل مستخدم يعمل.
- [ ] تغيير كلمة مرور مستخدم يعمل.
- [ ] روابط المعاينة تفتح الموقع العام وليس مسارًا داخليًا في الأدمن.
- [ ] لا يوجد زر ظاهر لا يعمل في الصفحات الأساسية.

## 3. Public Site

- [ ] `npm run release-check` ينجح.
- [ ] `npm run quality` ينجح.
- [ ] `npm run build` ينجح.
- [ ] الصفحة الرئيسية تعمل.
- [ ] صفحة الخدمات تعرض بيانات حقيقية.
- [ ] تفاصيل الخدمة تعمل.
- [ ] صفحة الأنظمة تعرض بيانات حقيقية.
- [ ] تفاصيل النظام تعرض features/images/faqs/requirements.
- [ ] صفحة التطبيقات تعرض بيانات حقيقية.
- [ ] تفاصيل التطبيق تعرض زر تحميل عندما يوجد رابط تحميل.
- [ ] صفحة الأعمال تعرض بيانات حقيقية.
- [ ] تفاصيل العمل تعرض gallery_images/technologies.
- [ ] نموذج التواصل يرسل ويظهر في الأدمن.
- [ ] نموذج طلب عرض السعر يرسل ويظهر في الأدمن.
- [ ] نموذج الدعم يرسل ويظهر في الأدمن.
- [ ] العربية RTL لا تنكسر.
- [ ] الإنجليزية LTR لا تنكسر.
- [ ] الثيم الفاتح مقبول بصريًا.
- [ ] الثيم الداكن مقبول بصريًا.
- [ ] التصميم responsive على الموبايل والتابلت والديسكتوب.

## 4. Content Rules

- [ ] لا يظهر محتوى تجريبي في الإنتاج إلا إذا تم تفعيله صراحة.
- [ ] لا تظهر عبارات تقنية للزائر مثل `coming_soon` أو `cross_platform` بدون تسمية مفهومة.
- [ ] لا يظهر زر "فتح التطبيق" كتصرّف أساسي عندما يوجد رابط تحميل مباشر.
- [ ] الهوية العامة تدار من الإعدادات وليست مكررة يدويًا داخل كل صفحة.

## 5. Clean Delivery

- [ ] لا يوجد `node_modules` داخل نسخة التسليم.
- [ ] لا يوجد `.next` داخل نسخة التسليم.
- [ ] لا يوجد `__pycache__` أو `*.pyc`.
- [ ] لا يوجد `db.sqlite3`.
- [ ] لا يوجد `uploads` أو `media` تجريبية.
- [ ] لا يوجد ملف `.env` حقيقي.
- [ ] توجد ملفات `.env.example` واضحة.
- [ ] يوجد `plan.md` محدث.
- [ ] توجد تقارير المراحل داخل `docs`.

## 6. Command Summary

cd D:\platform
.\scripts\full-e2e-check.ps1
```

للفحص السريع بدون build ثقيل:

```
