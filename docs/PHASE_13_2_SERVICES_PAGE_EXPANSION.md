# Phase 13.2 — Services Page Professional Expansion

## الهدف
تحويل صفحة خدماتنا من قائمة بسيطة إلى صفحة تسويقية احترافية تحتوي على كروت بصرية لكل خدمة، مع إبراز المميزات أسفل العنوان، وإمكانية إضافة/تعديل الخدمات من لوحة التحكم.

## ما تم

- إضافة كرت بصري مخصص للخدمات `PublicServiceCard`.
- عرض المميزات داخل كرت الخدمة مباشرة.
- دعم صورة الخدمة أو أيقونة مركزية بديلة.
- إضافة زرين لكل كرت: تفاصيل الخدمة وابدأ مشروعك.
- إضافة خدمات افتراضية احترافية تظهر عندما لا توجد خدمات مضافة من لوحة التحكم بعد.
- إبقاء الخدمات الديناميكية من لوحة التحكم هي المصدر الأساسي عند وجودها.
- تطوير صفحة تفاصيل الخدمة لتدعم الخدمات القادمة من الأدمن والخدمات الافتراضية.
- إضافة قسم ثابت في صفحة الخدمات يشرح طريقة العمل.
- إضافة مراحل تنفيذ داخل صفحة تفاصيل الخدمة.
- توسيع إدارة الخدمات في لوحة التحكم لدعم:
  - الوصف الكامل عربي/إنكليزي.
  - الأيقونة.
  - الصورة عبر Media Picker.
  - قائمة المميزات.
  - بيانات SEO.
- تحديث API العام للخدمات ليعيد مميزات الخدمة في صفحة القائمة أيضًا.

## الخدمات الافتراضية المضافة للعرض

1. تطوير مواقع الويب.
2. تطوير تطبيقات الويب.
3. تطوير لوحات التحكم.
4. تطوير تطبيقات سطح المكتب.
5. تطوير الأنظمة الإدارية.
6. ربط الأنظمة وواجهات API.
7. تحسين المشاريع القائمة.
8. الصيانة والدعم التقني.

## ملاحظات مهمة

- لا يوجد تغيير في مسارات الصفحات.
- لا يوجد Migration جديد لأن جدول مميزات الخدمات موجود مسبقًا.
- إذا أضاف الأدمن خدمات من لوحة التحكم، تختفي الخدمات الافتراضية ويظهر محتوى الأدمن.
- الهدف من الخدمات الافتراضية أن لا تبدو الصفحة فارغة أثناء التجهيز الأولي للموقع.
