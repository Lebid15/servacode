# Phase 15.12 — Admin Requests & Customers Closure

## الهدف
إغلاق قسم **الطلبات والعملاء** داخل لوحة التحكم ليصبح مركزًا تشغيليًا واضحًا بدل مجرد تبويبات منفصلة.

## ما تم تنفيذه

### Backend
- إضافة endpoint جديد:
  - `GET /api/v1/admin/customers-overview`
- يلخص القسم بالكامل:
  - عدد العملاء المعروفين بناءً على البريد أو الهاتف.
  - طلبات المشاريع.
  - رسائل التواصل.
  - طلبات الدعم.
  - آراء العملاء.
  - العناصر المفتوحة.
  - العناصر العاجلة.
  - العناصر التي تنتظر العميل.
  - أحدث العناصر الواردة.
  - مواعيد المتابعة لطلبات المشاريع.
  - تنبيهات تشغيلية مباشرة.
- إضافة تصدير CSV لطلبات الدعم:
  - `GET /api/v1/admin/export/support-requests`

### Admin Panel
- تحويل صفحة `/admin/customers` من redirect إلى مركز فعلي.
- إضافة Component جديد:
  - `AdminCustomersOverviewPage.tsx`
- إضافة تبويب "نظرة عامة" داخل `AdminCustomerTabs`.
- تحديث Navigation ليظهر قسم الطلبات والعملاء كقسم واضح مع تبويبات عملية.
- تحديث Admin API client لدعم:
  - `getAdminCustomersOverview`
  - أنواع بيانات overview الخاصة بالطلبات والعملاء.
- تحديث release-check للتأكد من وجود المركز وعدم الرجوع للـ redirect.

## قرار المنتج
- الطلبات لا تُضاف يدويًا بشكل افتراضي؛ تأتي من الموقع العام.
- التطبيقات والأعمال الحقيقية وآراء العملاء الموثقة تبقى يدوية لأنها تحتاج بيانات وصور وروابط حقيقية.
- قسم الطلبات والعملاء يجب أن يكون مركز متابعة وتشغيل، وليس مجرد قائمة رسائل.
