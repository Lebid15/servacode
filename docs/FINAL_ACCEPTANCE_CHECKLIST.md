# Final Acceptance Checklist

## الهدف

هذه القائمة هي معيار قبول المشروع قبل اعتباره مغلقًا وجاهزًا لمرحلة النشر لاحقًا.
لا يتم الانتقال للنشر قبل إغلاق جميع البنود الحرجة.

## 1. Backend

- [ ] `python -m compileall config platform_api scripts tests` ينجح.
- [ ] `python scripts\quality_check.py` ينجح.
- [ ] `pytest` ينجح.
- [ ] إنشاء مستخدم جديد من API يحفظ كلمة المرور مشفرة.
- [ ] لا يظهر `hashed_password` في أي response.
- [ ] أخطاء 401/404/400 ترجع `success=false`.
- [ ] الخدمات تحفظ `features` وتعرضها.
- [ ] الأنظمة تحفظ `features/images/faqs` وتعرضها.
- [ ] التطبيقات تحفظ روابط التحميل وملفات التحميل.
- [ ] الأعمال تحفظ التقنيات ومعرض الصور.
- [ ] إعدادات الموقع تحفظ وتظهر في public settings.
- [ ] نماذج التواصل وطلب السعر والدعم تصل إلى لوحة الأدمن.
- [ ] Audit logs تظهر عند عمليات الإدارة الأساسية.

## 2. Admin Panel

- [ ] `pnpm run release-check` ينجح.
- [ ] `pnpm run quality` ينجح.
- [ ] `pnpm run build` ينجح.
- [ ] تسجيل الدخول يعمل.
- [ ] Dashboard يعرض بيانات حقيقية.
- [ ] إضافة/تعديل/إخفاء خدمة يعمل.
- [ ] إضافة/تعديل/إخفاء نظام يعمل.
- [ ] إضافة/تعديل/إخفاء تطبيق يعمل.
- [ ] إضافة/تعديل/إخفاء عمل يعمل.
- [ ] صفحة المستخدمين تنشئ مستخدمًا بكلمة مرور صحيحة.
- [ ] صفحة الأدوار والصلاحيات تعمل أو تكون مخفية إذا غير مكتملة.
- [ ] صفحة الإعدادات تحفظ الهوية العامة.
- [ ] مكتبة الوسائط ترفع وتختار الملفات بدون كسر.
- [ ] روابط المعاينة تفتح الموقع العام وليس لوحة الأدمن.
- [ ] لا يوجد زر ظاهر لا يعمل.

## 3. Public Site

- [ ] `npm run release-check` ينجح.
- [ ] `npm run quality` ينجح.
- [ ] `npm run build` ينجح.
- [ ] الصفحة الرئيسية تعمل.
- [ ] صفحة الخدمات تعرض بيانات حقيقية.
- [ ] صفحة تفاصيل الخدمة تعمل.
- [ ] صفحة الأنظمة تعرض الصور والمميزات والأسئلة.
- [ ] صفحة التطبيقات تعرض زر تحميل مباشر عند وجود رابط تحميل.
- [ ] صفحة الأعمال تعرض المعرض والتقنيات.
- [ ] صفحة التواصل ترسل البيانات.
- [ ] صفحة طلب عرض سعر ترسل البيانات.
- [ ] صفحة الدعم ترسل البيانات.
- [ ] لا يظهر محتوى تجريبي إلا إذا كان `NEXT_PUBLIC_SHOW_DEMO_CONTENT=true`.
- [ ] اللغة العربية تعمل RTL.
- [ ] اللغة الإنجليزية تعمل LTR.
- [ ] الثيم الفاتح والداكن يعملان.
- [ ] التصميم متجاوب على الموبايل والتابلت والديسكتوب.

## 4. Files and Security

- [ ] لا توجد ملفات `.env` حقيقية داخل النسخة.
- [ ] لا توجد قواعد بيانات SQLite داخل النسخة.
- [ ] لا توجد `node_modules` داخل النسخة.
- [ ] لا توجد `.next` أو build artifacts داخل النسخة.
- [ ] لا توجد uploads أو media runtime داخل النسخة.
- [ ] `.gitignore` يغطي الملفات الحساسة والمولدة.
- [ ] `README.md` واضح ومحدث.
- [ ] `plan.md` محدث.
- [ ] وثائق المراحل موجودة داخل `docs`.

## 5. قرار القبول

إذا نجحت البنود أعلاه، تعتبر النسخة:

```text
Release Candidate محلية جاهزة للانتقال لاحقًا إلى خطة النشر.
```

إذا فشل أي بند حرج، لا يتم النشر ولا يتم اعتبار المشروع مغلقًا.
